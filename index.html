Media Library

Images and PDF documents (scans, certificates, forms) both live here. Uploaded files are stored in your browser — keep them under a few MB each to avoid filling localStorage. Anything added by URL (🔗) stays linked to wherever it's hosted instead, so there's no size limit on those — hover a linked item and click ⬇️ to pull a permanent local copy (also updates any page already using it), useful if the original source might disappear. "⬇️ Download all" bundles every externally-linked image and document into one zip (an images/ and a documents/ folder) and immediately relinks them to those paths — upload both folders to your server before publishing, or the media will 404 until you do. Both actions need the source server to allow cross-origin downloads; when it doesn't, they automatically retry through a public CORS proxy (allorigins.win) as a fallback — your media briefly transits that third-party service in that case, fine for public photos but worth knowing. Insert an image into a page with 🖼️ in the editor toolbar, or link to a document with 📎.

Members

Mark a category "Members only" in Categories to hide it from logged-out visitors — nav, search, and listings all exclude it, and direct links prompt for login instead of showing content.

⚠️ This is a soft gate, not real security. content.json (including member-only page content and this member list) is downloaded in full by every visitor's browser — hiding it in the UI doesn't stop anyone who inspects page source or fetches content.json directly. Don't put anything genuinely sensitive behind it.

Adds them to the member list (published like everything else — publish for the login to actually work) and drafts an invite email with their credentials.

Import from WordPress

Upload a WordPress export file (Tools → Export → All content, in wp-admin) to bring its pages and posts in as local drafts/pages here. Nothing is published automatically — review and use Submit for Publish as normal once you're happy. Images stay linked to their original URLs rather than being downloaded.

Submit for Publish

Edit content locally until you're happy. All of the site's content lives in a single content.json file — publishing just means replacing that file on the server. Use this screen to get your changes there, including any uploaded images.
